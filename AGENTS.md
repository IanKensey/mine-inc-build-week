# Mine Inc. — Build Week Instructions

## Project goal

Build a small playable Godot 4 demonstration of Mine Inc.

The player must be able to:

1. View an escape pod in 3D space.
2. Rotate and zoom the camera.
3. Observe an asteroid following a visible trajectory.
4. Select and fire at the asteroid.
5. Destroy the asteroid and release resources.
6. collect those resources.
7. Manufacture one module.
8. Attach that module to the escape pod on a 3D grid.

## Scope constraints

- Use Godot 4 and typed GDScript.
- Target Windows 11.
- Do not implement pirates, research, power routing, trading, mobile support, campaign content, or sophisticated orbital physics.
- Prefer simple placeholder geometry over external assets.
- Keep game data configurable rather than hard-coded where practical.
- Keep every commit runnable.
- Do not add third-party dependencies without approval.
- Avoid copyrighted artwork, music, names, or visual designs.

## Engineering practices

- Use clear node and file names.
- Add type annotations to GDScript.
- Keep scripts focused and reasonably small.
- Add comments for non-obvious logic.
- Log meaningful state changes.
- Test each feature in Godot after implementation.
- Update README.md when setup or controls change.
- Update docs/codex-collaboration-log.md after significant tasks.

## Visual direction

Use clean, readable placeholder 3D geometry with a simple industrial science-fiction feel. Prioritise clarity and function over polished art.
